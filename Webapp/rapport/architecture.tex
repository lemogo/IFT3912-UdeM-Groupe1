	\section*{Descritption}
	\indent{L'architecture globale du projet est composée de trois modules principals à savoir le Module de gestion des pages html, le module de base de donnée et le module pont de gestionnaire de l'application}
	
	\section{Module Gestionnaire des pages html}
		Le module gestionnaire des pages html traite des requêtes effectué avec le serveur HTTP, parconséquent il intéragit avec Jetty. Ce 			  module est chargé de gérer les pages html statiques et d'opérer la logique dynamique des pages html. Les classes de génération de 		    page thml devront communiquer avec des classes du module pont Gestionnaire de l'application afin d'obtenir les information               necessaires à issues des requêtes pour la génération dynamiques des pages html. Par ailleurs aucune requête ne devra être fait dans ce     module. le lien avec les classes du pont gestionnaire de l'application se feront par des instances ou objets de ces classes du pont       			    Gestionnaire. 
	
		\section{Module Pont gestionnaire d l'application}
		    Ce module comporte des classes spécialisées qui contiennent  des informations de la base de données ainsi des classes de                   gestion des usagers des comptes et des évènements. Ce module est le lien entre les pages html et la base de donnée il devra donc          communiquer avec les deux autres modules afin de récupérer les informations demandées de la base de donnée ensuite les traiter 					et les soumettres aux gestionnaire des pages html. 
		    	
		\section{Module Base de données}
		 	ce module est charger de fournir toutes les informations venant des requêtes HTTP et qui necessitent l'utilisation des données de   			la base. Lotalité de requêtes seront exclusivement programmées dans ce module. Aucun autre module ne devra contenir des requête de         connexion à la base. Le résultat des  requetes sera mis à la dispositions des classes du pont  gestionnaire de l'application ainsi 				le modune base de données communiquera avec le module pont gestionnaire d'application par instanciationtion de ses classes.
		 
		    
		  \section{Choix de conception}
		  	Nous avons choisi cette division modulaire dans le souci de séparation adéquates des tâches et de la spécialisation selon les 						niveaux de traitement dans l'application. Ainsi nous avons choisi les classes pont destionnaires d'application pour éviter de 						surcharger la base de données de flux de requêtes en suite la gestion des usager anonymes se fait par le pont gestionnaire. Les 					usagers connectés pourront avoir leur session et chacune sera représentée par des instances de classe du pont. Cest approche nous 				permet donc de faire une une subdivision spécialisées et cadre bien avec les différentes compétences des membres de l'équipe.
